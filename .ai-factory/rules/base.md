# Project Base Rules

> Auto-detected conventions from codebase analysis. Edit as needed.

## Naming Conventions

- **PHP files / classes:** PascalCase, one class per file, PSR-4 (`App\` → `app/`). Example: `ProfileController`, `PasswordValidationRules`.
- **PHP methods / variables:** camelCase, descriptive intent-revealing names (`isRegisteredForDiscounts`, not `discount()`).
- **PHP enum keys:** TitleCase.
- **TypeScript/React files:** kebab-case (`app-sidebar.tsx`, `two-factor-setup-modal.tsx`).
- **React components / types:** PascalCase exports; hooks are `useXxx` camelCase.
- **Routes:** dot-namespaced named routes (`profile.edit`, `security.edit`); always link via `route()` (PHP) or Wayfinder helpers (TS), never hardcoded URLs.

## Module Structure

- **Backend (`app/`):** `Actions/Fortify/` (auth actions), `Concerns/` (shared validation traits), `Http/Controllers/`, `Http/Middleware/`, `Http/Requests/`, `Models/`, `Providers/`.
- **Routes (`routes/`):** `web.php` (public + dashboard), `settings.php` (profile/security/appearance), `console.php`. Auth routes are registered by Fortify, not declared here.
- **Frontend (`resources/js/`):** `pages/` (Inertia page components, 1:1 with `Inertia::render()` strings), `components/` + `components/ui/` (shadcn-style primitives), `layouts/`, `hooks/`, `lib/`, `types/`. `routes/` and `actions/` are **generated by Wayfinder** — never hand-edit.
- **Tests (`tests/`):** `Feature/` (primary), `Unit/`; Pest.
- Do not create new top-level directories without approval; follow the existing structure.

## Error Handling

- Validate input with dedicated Form Requests (`app/Http/Requests/**`) and shared rule traits in `app/Concerns/`; keep rules defined once and reused by both Fortify actions and requests.
- Fail with descriptive, structured responses; do not silently swallow exceptions.

## Logging

- Laravel logging via `config/logging.php`. Tail logs in development with `php artisan pail` (`laravel/pail`).

## Testing

- Every change must be programmatically tested (Pest). Prefer feature tests; use model factories and existing factory states.
- Run the minimum scope: `php artisan test --compact --filter=Name`.

## Formatting & Static Analysis

- **PHP:** `vendor/bin/pint --dirty` (Laravel preset) before finalizing; `vendor/bin/phpstan analyse` (Larastan, `phpstan.neon`) must pass.
- **TypeScript:** strict mode; imports use the `@/*` → `resources/js/*` alias. Run `npm run lint`, `npm run format`, `npm run types:check`. Prettier + `prettier-plugin-tailwindcss` order Tailwind classes.
