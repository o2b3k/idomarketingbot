# Development

Day-to-day commands and conventions for working on `idomarketing`.

## Command Reference

| Task | Command |
|------|---------|
| Start dev environment (server + queue + logs + Vite) | `composer run dev` |
| Full local gate (config clear → Pint → PHPStan → tests) | `composer run test` |
| CI gate (adds ESLint + Prettier + tsc) | `composer run ci:check` |
| Run all tests | `php artisan test --compact` |
| Run one test file / filter | `php artisan test --compact --filter=TestName` |
| Format PHP (only changed files) | `vendor/bin/pint --dirty` |
| Static analysis (PHP) | `vendor/bin/phpstan analyse` |
| Frontend build | `npm run build` |
| Frontend typecheck | `npm run types:check` |
| Frontend lint / format | `npm run lint` · `npm run format` |
| Regenerate Wayfinder route helpers | `php artisan wayfinder:generate` |

## Testing

Tests use **Pest**. Feature tests are primary and live in `tests/Feature/`; unit tests in `tests/Unit/`.

- Every change must be covered by a test — write or update one, then run it.
- Create tests with `php artisan make:test --pest SomeFeatureTest` (omit the `Feature/` prefix).
- Use model factories (and their states) rather than manual model setup.
- Run the minimum scope during development:

```bash
php artisan test --compact --filter=ProfileUpdate
```

See [Testing conventions in the base rules](../.ai-factory/rules/base.md).

## Formatting & Static Analysis

**PHP** — always run Pint on changed files before finalizing, and keep PHPStan green:

```bash
vendor/bin/pint --dirty      # Laravel preset
vendor/bin/phpstan analyse   # Larastan, config: phpstan.neon
```

**TypeScript / React** — strict mode is on; imports use the `@/*` → `resources/js/*` alias:

```bash
npm run lint          # ESLint (auto-fix)
npm run format        # Prettier (+ prettier-plugin-tailwindcss orders Tailwind classes)
npm run types:check   # tsc --noEmit
```

## Conventions

- **PHP files/classes:** PascalCase, PSR-4 (`App\` → `app/`); methods/variables camelCase with intent-revealing names.
- **TypeScript files:** kebab-case (`app-sidebar.tsx`); components export PascalCase.
- **Routes:** use named routes via `route()` (PHP) or Wayfinder helpers (TS) — never hardcode URLs.
- **New files the Laravel way:** use `php artisan make:*` generators with `--no-interaction`.
- Follow the existing structure; don't add top-level directories without approval.

Full details: [`.ai-factory/rules/base.md`](../.ai-factory/rules/base.md).

## Frontend Notes

- `resources/js/routes/` and `resources/js/actions/` are **generated by Wayfinder** (Vite plugin in dev, or `php artisan wayfinder:generate`). Never edit them by hand.
- If a change doesn't appear in the browser, ensure Vite is running (`composer run dev`) or run `npm run build`.
